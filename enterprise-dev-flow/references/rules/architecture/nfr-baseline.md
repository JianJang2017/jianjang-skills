# 非功能性需求基线

> 适用范围：所有系统设计和开发，适用于产品经理、架构师、研发人员

---

## 1. 规则说明

非功能性需求（Non-Functional Requirements，NFR）定义了系统的质量属性，如性能、可用性、安全性等。明确的 NFR 基线能够确保系统满足业务要求，避免上线后出现性能或可用性问题。

## 2. 规则内容

### 2.1 性能指标

| 指标 | 基线要求 | 说明 |
|------|---------|------|
| P99 响应时间 | < 200ms | 99% 的请求响应时间小于 200ms |
| QPS 峰值 | 根据业务评估 | 系统能够承受的每秒请求数 |
| 并发用户数 | 根据业务评估 | 系统能够同时支持的在线用户数 |
| 数据库连接数 | < 100 | 单个服务的数据库连接数 |
| 线程池大小 | 根据压测结果 | 合理设置线程池大小，避免资源浪费 |

**核心接口性能要求：**
- 查询接口：P99 < 200ms
- 创建接口：P99 < 500ms
- 更新接口：P99 < 500ms
- 批量操作：P99 < 2s

### 2.2 可用性

| 指标 | 基线要求 | 说明 |
|------|---------|------|
| SLA 目标 | 99.99% | 年度可用时间 99.99%（约 52 分钟故障时间） |
| 降级方案 | 必须 | 核心功能必须有降级方案 |
| 熔断阈值 | 错误率 > 50% | 触发熔断，保护系统 |
| 限流策略 | 根据 QPS 设置 | 防止系统过载 |
| 回滚计划 | 必须 | 上线失败时的回滚方案 |

**降级策略：**
- 非核心功能降级：如推荐、统计等
- 读降级：返回缓存数据或默认值
- 写降级：异步处理或延迟处理

### 2.3 安全性

| 指标 | 基线要求 | 说明 |
|------|---------|------|
| 认证方式 | OAuth2 / JWT | 统一认证方式 |
| 加密策略 | HTTPS / AES | 传输加密 + 存储加密 |
| 频率限制 | 根据接口设置 | 防止恶意攻击 |
| 敏感信息脱敏 | 必须 | 手机号、身份证号等 |
| 权限控制 | RBAC / ABAC | 角色权限控制 |

**安全基线：**
- 所有接口必须使用 HTTPS
- 敏感信息必须加密存储
- 密码必须使用 BCrypt 加密
- 接口必须有认证和授权
- 防止 SQL 注入、XSS 注入

### 2.4 可观测性

| 指标 | 基线要求 | 说明 |
|------|---------|------|
| 日志 TraceID | 必须 | 全链路追踪 |
| 监控告警阈值 | 根据业务设置 | 及时发现问题 |
| 埋点 | 核心流程必须 | 业务数据分析 |
| 日志级别 | INFO / WARN / ERROR | 合理设置日志级别 |
| 日志保留时间 | 30 天 | 日志保留时间 |

**日志规范：**
- 关键操作必须记录日志
- 日志必须包含 TraceID
- 日志不能包含敏感信息
- 异常必须记录堆栈信息

### 2.5 容灾

| 指标 | 基线要求 | 说明 |
|------|---------|------|
| 数据备份 | 每天 | 数据库每天备份 |
| 备份保留时间 | 30 天 | 备份保留时间 |
| 灾难恢复时间（RTO） | < 30 分钟 | 系统恢复时间 |
| 数据恢复点（RPO） | < 1 小时 | 数据丢失时间 |
| 多机房部署 | 推荐 | 提高可用性 |

## 3. 示例

### 正确示例

**性能指标定义：**
```
功能：用户登录
性能要求：
- P99 响应时间：< 200ms
- QPS 峰值：1000
- 并发用户数：10000
- 压测结果：P99 = 150ms，QPS = 1200
```

**可用性设计：**
```
功能：订单查询
可用性要求：
- SLA 目标：99.99%
- 降级方案：查询失败时返回缓存数据
- 熔断阈值：错误率 > 50%，熔断 30 秒
- 限流策略：单用户 QPS 限制为 10
```

**安全性设计：**
```
功能：用户登录
安全性要求：
- 认证方式：JWT
- 加密策略：HTTPS + BCrypt
- 频率限制：单 IP 每分钟最多 10 次
- 敏感信息脱敏：手机号显示为 138****1234
- 权限控制：RBAC
```

**可观测性设计：**
```
功能：订单创建
可观测性要求：
- 日志 TraceID：必须
- 监控告警：订单创建失败率 > 1% 时告警
- 埋点：记录订单创建成功/失败次数
- 日志级别：INFO
- 日志内容：订单 ID、用户 ID、金额、状态
```

### 错误示例

```
# ❌ 没有定义性能指标
功能：用户登录
性能要求：尽可能快

# ❌ 没有降级方案
功能：订单查询
可用性要求：SLA 目标 99.99%

# ❌ 没有加密策略
功能：用户登录
安全性要求：需要认证

# ❌ 没有 TraceID
功能：订单创建
可观测性要求：记录日志
```
