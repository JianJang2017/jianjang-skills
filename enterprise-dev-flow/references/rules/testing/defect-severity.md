# 缺陷等级标准

> 适用范围：所有测试发现的缺陷分级，适用于测试工程师、产品经理、研发人员

---

## 1. 规则说明

统一的缺陷等级标准能够帮助团队快速评估问题的严重程度，合理安排修复优先级。本规范定义了缺陷的分级标准、响应时效和典型示例。

## 2. 规则内容

### 2.1 缺陷等级定义

| 等级 | 定义 | 研发响应时效 | 典型示例 |
|------|------|------------|---------|
| **P0（致命）** | 系统崩溃、数据丢失、核心流程阻断 | 2小时内修复 | 登录接口500错误，支付后钱扣了订单没生成 |
| **P1（严重）** | 主要功能错误，无临时解决方案 | 24小时内修复 | 支付成功但订单状态未更新 |
| **P2（一般）** | 次要功能错误，有临时方案或影响较小 | 版本发布前修复 | 页面样式错乱，非核心字段显示错误 |
| **P3（轻微）** | 建议性问题、文案错误 | 视资源情况排期 | 提示语歧义，UI微调建议 |

### 2.2 P0（致命）缺陷

**定义：**
- 系统崩溃，无法启动或运行
- 核心业务流程完全阻断，无法继续
- 数据丢失或数据损坏
- 严重的安全漏洞

**典型示例：**
- 登录接口返回 500 错误，所有用户无法登录
- 支付接口异常，用户支付后钱扣了但订单没生成
- 数据库连接失败，系统无法访问
- SQL 注入漏洞，可以获取所有用户数据
- 系统内存溢出，服务频繁重启

**响应时效：**
- 2 小时内修复
- 必要时回滚到上一个稳定版本

### 2.3 P1（严重）缺陷

**定义：**
- 主要功能错误，影响核心业务
- 无临时解决方案或绕过方式
- 影响大量用户使用

**典型示例：**
- 支付成功但订单状态未更新为"已支付"
- 用户注册后无法收到验证码
- 订单列表查询超时，无法查看订单
- 文件上传失败，无法提交表单
- 权限控制失效，普通用户可以访问管理员功能

**响应时效：**
- 24 小时内修复
- 如果无法及时修复，需要提供临时解决方案

### 2.4 P2（一般）缺陷

**定义：**
- 次要功能错误，不影响核心业务
- 有临时解决方案或绕过方式
- 影响较小或影响少量用户

**典型示例：**
- 页面样式错乱，但不影响功能使用
- 非核心字段显示错误（如创建时间格式不对）
- 搜索功能不支持模糊查询
- 导出功能偶尔失败，重试可以成功
- 提示信息不够友好

**响应时效：**
- 版本发布前修复
- 可以延后到下一个版本修复

### 2.5 P3（轻微）缺陷

**定义：**
- 建议性问题，不影响功能使用
- 文案错误、UI 微调建议
- 体验优化建议

**典型示例：**
- 提示语歧义或错别字
- 按钮位置不够美观
- 页面加载动画不够流畅
- 字段排序不够合理
- 建议增加某个快捷操作

**响应时效：**
- 视资源情况排期
- 可以延后到后续版本修复

## 3. 示例

### 正确示例

**P0 缺陷报告：**
```
缺陷标题：登录接口返回 500 错误，所有用户无法登录
缺陷等级：P0（致命）
缺陷描述：
1. 访问登录页面，输入用户名和密码
2. 点击"登录"按钮
3. 接口返回 500 错误，提示"系统繁忙，请稍后重试"
4. 所有用户都无法登录，核心业务完全阻断

影响范围：所有用户
重现概率：100%
期望修复时间：2 小时内
```

**P1 缺陷报告：**
```
缺陷标题：支付成功但订单状态未更新为"已支付"
缺陷等级：P1（严重）
缺陷描述：
1. 创建订单，订单状态为"待支付"
2. 点击"立即支付"，完成支付
3. 支付成功，但订单状态仍然是"待支付"
4. 用户无法查看订单详情，影响后续发货流程

影响范围：所有支付用户
重现概率：50%
期望修复时间：24 小时内
```

**P2 缺陷报告：**
```
缺陷标题：订单列表创建时间格式显示错误
缺陷等级：P2（一般）
缺陷描述：
1. 访问订单列表页面
2. 查看订单的创建时间
3. 创建时间显示为"2024-03-23T10:30:00"，应该显示为"2024-03-23 10:30:00"

影响范围：所有用户
重现概率：100%
期望修复时间：版本发布前
```

**P3 缺陷报告：**
```
缺陷标题：提示语存在错别字
缺陷等级：P3（轻微）
缺陷描述：
1. 访问用户设置页面
2. 查看提示语
3. 提示语为"请输入正确的手机号吗"，应该为"请输入正确的手机号"

影响范围：所有用户
重现概率：100%
期望修复时间：视资源情况排期
```

### 错误示例

```
# ❌ 缺陷等级评估错误
缺陷标题：提示语存在错别字
缺陷等级：P0（致命）  # 应该是 P3

# ❌ 缺陷描述不清晰
缺陷标题：登录有问题
缺陷等级：P1（严重）
缺陷描述：登录不了  # 缺少详细的重现步骤

# ❌ 没有说明影响范围
缺陷标题：支付失败
缺陷等级：P1（严重）
缺陷描述：支付的时候失败了  # 缺少影响范围和重现概率
```

## 4. 检查清单

- [ ] 缺陷等级评估是否准确？
- [ ] 缺陷描述是否清晰？
- [ ] 是否包含详细的重现步骤？
- [ ] 是否说明了影响范围？
- [ ] 是否说明了重现概率？
- [ ] 是否说明了期望修复时间？
- [ ] P0 缺陷是否立即通知相关人员？
- [ ] P1 缺陷是否在 24 小时内响应？
