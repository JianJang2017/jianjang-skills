## 📝 规则文件特点

每个规则文件都包含：

1. **标题和适用范围** - 明确规则的适用场景
2. **规则说明** - 详细的规则说明和背景
3. **规则内容** - 具体的规则条目和标准
4. **示例** - 正确示例和错误示例对比
5. **检查清单** - 便于自查的检查项
6. **版本信息** - 版本号和最后更新时间

### 示例格式

```markdown
# [规则名称]

> 适用范围：[说明适用于哪些场景]

---

## 1. 规则说明
[详细的规则说明]

## 2. 规则内容
[具体的规则条目]

## 3. 示例

### 正确示例
[展示正确的做法]

### 错误示例
[展示错误的做法]

## 4. 检查清单
- [ ] 检查项1
- [ ] 检查项2

---

**版本：** 1.0
**最后更新：** 2026-03-23
```

**核心要点：**
- 敏感信息保护（密码、手机号、身份证等必须脱敏）
- SQL 安全（禁止 SQL 注入）
- 越权访问防护（必须校验权限）
- 错误信息安全（不暴露堆栈信息）

**详细规范参考：** `../rules/security/security-red-lines.md`
**敏感数据处理：** `../rules/security/sensitive-data.md`

## 📚 规则分类说明

### security/ - 安全规则
保护系统和数据安全的核心规则，包括敏感信息保护、SQL安全、越权防护等。

### database/ - 数据库规则
数据库设计和使用规范，包括表命名、字段定义、索引设计、SQL编写规范等。

### api/ - 接口规则
API接口设计规范，包括RESTful设计、响应格式、URL命名、参数校验、分页、错误码等。

### code-quality/ - 代码质量规则
代码质量保障规则，包括幂等性设计、状态机规范、事务边界控制等。

### git/ - Git规则
代码版本管理规范，包括提交信息格式、分支命名、提交前检查等。

### testing/ - 测试规则
测试相关规范，包括缺陷等级标准、验收标准格式、测试覆盖矩阵等。

### architecture/ - 架构规则
系统架构设计规范，包括非功能性需求基线、缓存策略、消息队列设计等。
