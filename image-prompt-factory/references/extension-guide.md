# 扩展指南：如何添加新风格

本指南说明如何向 Image Prompt Factory 技能添加新的风格分类。

## 添加流程

### 步骤1: 创建风格参考文件

在 `references/` 目录下创建新文件,命名格式: `XX-风格名称.md`

**文件结构模板**:

```markdown
# [风格名称]风格参考

## 风格特征

用2-3段文字描述该风格的核心特点:
- 视觉特征
- 文化背景
- 典型应用场景
- 与其他风格的区别

## 专属要素

### 1. [要素类型1]
列出该风格特有的第一类要素,如:
- 子要素A: 描述
- 子要素B: 描述

### 2. [要素类型2]
列出第二类专属要素...

### 3. [要素类型3]
继续列出...

(通常3-5个专属要素类型即可)

## 常用术语

### [术语分类1]
```
关键词1, 关键词2, 关键词3,
术语A, 术语B, 术语C
```

### [术语分类2]
```
专业词汇列表
```

## 典型Prompt片段示例

### 人物描述
```
该风格典型的人物描述示例
保持2-4行,展示风格特色
```

### [其他关键部分]
```
服装/场景/光线等典型描述片段
每个片段2-4行即可
```

## 画幅建议
- 用途A: 推荐画幅
- 用途B: 推荐画幅

## 光线建议
- 场景A: 光线类型
- 场景B: 光线类型

## 注意事项
- 该风格需要特别注意的点1
- 该风格需要特别注意的点2
- 常见错误和避免方法
```

### 步骤2: 在SKILL.md中注册新风格

打开 `SKILL.md`,在"支持的风格分类"部分添加:

```markdown
X. **[新风格名称]** - 简短描述(1行,20字内)
```

同时在"风格定位与参考"部分添加文件映射:

```markdown
- `references/XX-风格名称.md` - [新风格名称]
```

### 步骤3: 测试新风格

创建测试用例,确保:
1. 技能能正确识别该风格关键词
2. 能读取并应用风格参考文件
3. 生成的prompt符合风格特征
4. 专属要素被正确提取

**测试示例**:
```
用户: "帮我生成一个[新风格]的图片提示词,[具体需求描述]"

预期: 
- 技能识别风格为"[新风格]"
- 读取 references/XX-风格名称.md
- 应用该风格的专属要素
- 输出符合该风格特征的标准prompt
```

## 风格参考文件编写要点

### 1. 风格特征部分
- **明确边界**: 说清楚这个风格"是什么"和"不是什么"
- **突出差异**: 与相似风格的区别点
- **实用导向**: 聚焦可操作的视觉特征

**好的示例**:
```
韩系CCD风格是模仿早期数码相机拍摄效果的现代时尚摄影,
特点是冷白色调、轻微过曝、CCD特有颗粒感和夜间生活氛围。
与日系胶片风的区别在于更注重数码质感而非胶片温润感。
```

**避免模糊**:
```
❌ 这是一种很好看的风格
❌ 这个风格很流行
❌ 这种风格有很多变化
```

### 2. 专属要素部分
- **精选3-5类**: 不要超过8类,保持聚焦
- **层次清晰**: 每类下列出3-8个具体子项
- **可操作性**: 每个要素都要能直接用于prompt

**好的示例**:
```
### CCD滤镜效果
- **冷白CCD**: 高色温、蓝白清透、轻颗粒
- **黑珍珠CCD**: 低饱和黑珍珠暗部、咖啡暖金反光
- **海军蓝CCD**: 背景海军蓝与黑曜黑、冷白直闪
```

### 3. 常用术语部分
- **中英对照**: 如果有英文术语,提供对照
- **分类组织**: 按主题分组,不要堆砌
- **高频优先**: 列出最常用的20-30个词

### 4. 典型片段示例
- **真实可用**: 从实际优质prompt中提取
- **保持简短**: 每个示例2-4行
- **展示特色**: 突出该风格的独特表达方式

## 常见问题

### Q: 新风格与现有风格有重叠怎么办?

A: 
1. 如果重叠度>70%,考虑作为子分类合并到现有风格
2. 如果重叠度30-70%,在风格特征中明确区分点
3. 如果重叠度<30%,可以独立成新风格

### Q: 风格参考文件应该多详细?

A: 
- 最小版本: 1000字(风格特征+3个专属要素+基本术语)
- 标准版本: 1500-2000字(完整结构+充足示例)
- 不要超过: 3000字(保持精炼,避免信息过载)

### Q: 如何确保新风格被正确识别?

A: 在风格特征部分列出**触发关键词**:
```
## 触发关键词
用户说出以下词汇时应识别为本风格:
- 关键词1, 关键词2, 关键词3
- 短语A, 短语B
```

### Q: 风格参考文件可以更新吗?

A: 可以且应该。当发现:
- 新的典型表达方式
- 更准确的术语
- 更好的示例片段
- 用户反馈的改进点

随时更新文件,无需修改SKILL.md核心逻辑。

## 质量检查清单

新风格添加完成后,确认:
- [ ] 风格参考文件结构完整
- [ ] 风格特征描述清晰明确
- [ ] 专属要素3-5类,每类有具体子项
- [ ] 常用术语分类组织,不少于20个
- [ ] 典型片段示例真实可用
- [ ] 在SKILL.md中正确注册
- [ ] 通过至少3个测试用例
- [ ] 文件命名规范(XX-风格名.md)
- [ ] 与现有风格区分明确
- [ ] 示例长度适中(2-4行/个)

## 维护建议

建议每个风格参考文件每季度review一次:
1. 检查是否有过时术语
2. 补充新发现的典型表达
3. 根据用户反馈优化
4. 更新示例片段

保持技能的专业性和时效性。
