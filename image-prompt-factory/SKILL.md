# Image Prompt Factory

为 **GPT-image / DALL-E** 生成专业图片提示词的技能。

核心理念:**先在脑中把画面看清楚,再用自然语言流畅地描述出来**,而不是填充模板。

## 目标生成器

**GPT-image / OpenAI DALL-E 系列**

特性:
- ✅ 理解自然语言描述(段落式,非tag串)
- ✅ 中英文都支持,混合使用也可以
- ✅ 对场景感、故事性、内部逻辑响应好
- ❌ 不支持 negative prompt
- ❌ 不需要 SD 式咒语(`masterpiece, best quality`)
- ❌ 不需要 MJ 式参数(`--ar 9:16`)

## 触发条件

- "帮我生成一个XX风格的图片提示词"
- "我想要一张XX样子的照片"  
- "给我写个生图prompt,风格是XX"
- "参考这个风格,帮我写提示词"

7. **泳装/海边/度假** - 泳装写真、海滩度假、阳光海水
8. **3D CG/角色/幻想** - 游戏角色、3D渲染、次世代
9. **其他/生活写真** - 日常生活、纪实摄影、真实质感
10. **新中式/东方美学** - 茶室屏风、改良新中式、留白静气(≠古装剧)
11. **复古港风** - 旧港片剧照、霓虹侧光、低饱和胶片

18. **低调电影感** - 暗调静帧、可读暗部、叙事情绪

**风格分流提示**(易混淆的优先级):
- 茶室/屏风/留白/改良 → 优先 `新中式`,不要误判成古风仙侠
- 度假旅拍穿搭多样 → `旅行假日`;单聚焦泳装身体 → `泳装/海边`
- 松垮随性优雅氛围 → `法式慵懒`;贴身柔光强调线条 → `居家/柔光`;纪实抓拍 → `生活写真`
- 上传服装/主图/不要色差 → `电商试衣`,风格词只作画面辅助
- 怼脸未修图微纹理 → `超近景真实人脸`;普通近景精修头像不算
- 低照度+动机光源+可读暗部+叙事 → `低调电影感`;只有"暗调/电影感"字面词不算

## 核心工作流

### 第一步: 理解需求

分析用户描述:
- **风格方向**: 属于哪个分类?
- **核心诉求**: 人物、场景、氛围、情绪
- **具体细节**: 画幅、构图、光线偏好

**如果需求开放**(如"古风少女"):
- 给出 2 个不同方向的概念(如 A.春日庭院 / B.江南烟雨)
- 用户选择后再展开完整 prompt

**如果需求明确**(如"春日庭院里穿粉色汉服的古风少女"):
- 直接生成 1 个完整 prompt

需求模糊时简短提问:
- "更偏古典还是现代?"
- "室内还是户外?"
- "想要什么氛围?"
- "画幅有要求吗?"

### 第二步: 吸收风格DNA

**读取对应的风格参考文件**:
- `references/01-guofeng.md` - 古风/国风/仙侠
- `references/02-japanese.md` - 日系
- `references/03-korean-ccd.md` - 韩系/CCD
- `references/04-urban.md` - 都市/街拍
- `references/05-workplace.md` - 职场/学院
- `references/06-homewear.md` - 居家/柔光
- `references/07-swimwear.md` - 泳装/海边
- `references/08-3dcg.md` - 3D CG/幻想
- `references/09-lifestyle.md` - 生活写真
- `references/10-new-chinese.md` - 新中式/东方美学
- `references/11-hongkong-retro.md` - 复古港风
- `references/12-french-lazy.md` - 法式慵懒
- `references/13-travel-vacation.md` - 旅行假日

获取:
- 该风格的视觉特征
- 语言风格(中文/英文/混合)
- 源文件路径(含真实实例)

**读取源文件中的真实实例**:
从 `prompts/examples/XX-example.md` 读取 2-3 条同类实例,吸收:
- 用词习惯
- 描述节奏
- 长度感觉
- 细节粒度

### 第三步: 在脑中构建画面

**关键: 先"看见"画面,再动笔**

内部自检(不输出):
- [ ] **人物**: 年龄明确为成年,五官、气质、表情
- [ ] **服装**: 款式、材质、颜色、细节
- [ ] **姿态**: 站/坐/蹲,手部动作,眼神方向
- [ ] **场景**: 室内/户外,具体环境,前后景
- [ ] **光线**: 来源、方向、色温、效果
- [ ] **镜头**: 景别(特写/半身/全身)、视角
- [ ] **氛围**: 情绪、色调、质感
- [ ] **瞬间**: 这一刻在发生什么?有内部逻辑吗?

**安全检查**(强制):
- [ ] 年龄明确标注为成年(22-28岁等)
- [ ] 避免未成年相关描述
- [ ] 人体比例合理,手部解剖正确
- [ ] 不冒用真人姓名/身份
- [ ] 避免低俗、暴力、血腥

### 第四步: 用自然语言描述

**针对 GPT-image 的输出格式**:

```
【风格】[风格名称]
【画幅】[1024x1792 (9:16) / 1792x1024 (16:9) / 1024x1024 (1:1)]

【Prompt】
[自然流畅的段落式描述,2-4个段落]
[第1段: 场景+人物主体]
[第2段: 服装+妆容细节]
[第3段: 姿态+场景背景]
[第4段: 光线+镜头+质感]

[古风用中文,技术术语可用英文]
[长度: 150-400字,让画面决定,不要硬凑]
[保持内部逻辑: 光线→阴影, 姿势→表情, 材质→反光]

【要素拆解】(可选,用户要求时展开)
人物/服装/场景/光线/镜头等清单
```

**语言选择**:
- 古风/国风: 中文为主
- 日系: 中文,技术参数可用英文
- 韩系/CCD: 中英混合
- 都市/职场: 中文为主
- 3D CG: 英文为主,保留技术准确性
- 其他: 根据内容选择

**长度原则**:
- 简单场景: 150-250字
- 中等复杂: 250-350字
- 高度精细: 350-500字
- 不要为了凑字数堆砌形容词

**不要输出**:
- ❌ Negative prompt (GPT-image不支持)
- ❌ SD式咒语 (`masterpiece, best quality, 8k`)
- ❌ MJ式参数 (`--ar 9:16, --v 6`)
- ❌ 强制的8要素bullet清单(除非用户要求)

## 特殊情况处理

### 用户提供参考图时

1. 分析参考图的风格归属和视觉特征
2. 识别构图(景别、角度、比例)
3. 观察光线(色温、明暗、方向)
4. 提取色彩基调(主色、配色、饱和度)
5. 注意细节(服装、场景、质感、情绪)
6. 生成相似风格的新prompt

### 用户要求调整时

根据反馈精准调整:
- "更清冷一些" → 冷白/蓝灰色调,降低饱和度
- "不要这么性感" → 调整服装覆盖度,改变姿态描述
- "背景简单点" → 简化场景元素,增加虚化
- "加点故事感" → 添加动作、情绪、互动细节
- "给我要素拆解" → 展开人物/服装/场景等清单

### 多风格混合时

确定主导风格:
- 主导风格决定整体框架和语言
- 次要风格作为氛围补充
- 确保不产生冲突(需特殊处理时说明)

## 注意事项

### ⚠️ 内容安全(强制)
- ✅ 年龄明确标注为成年(22-28岁等具体年龄)
- ❌ 避免未成年相关描述
- ❌ 避免过度性暗示或低俗
- ❌ 不使用侵权角色名(用"原创"或"XX风格")
- ❌ 不涉及暴力、血腥、恐怖

### ⚠️ 质量原则
- ✅ 优先真实感,避免"AI感"、"塑料皮"
- ✅ 人体比例合理,手部解剖正确
- ✅ 细节丰富但不过度堆砌
- ✅ 色彩协调,避免冲突
- ✅ 描述要有内部逻辑(光线→阴影, 姿势→表情)

### ⚠️ 文化尊重
- 古风服饰遵循基本历史考据
- 尊重不同文化服饰传统
- 避免文化挪用或不当混搭

## 画幅规格参考

GPT-image 常用规格:
- **竖版人像**: 1024x1792 (9:16) - 全身/半身写真
- **方形**: 1024x1024 (1:1) - 特写/社交媒体
- **横版场景**: 1792x1024 (16:9) - 风景/宽幅构图
- **竖版特写**: 1024x1536 (2:3) - 半身到头部
