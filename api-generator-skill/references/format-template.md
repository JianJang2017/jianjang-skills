# 标准格式参考

这是生成 API 文档时必须遵循的完整格式模板。

## 完整示例（登录接口）

```markdown
#### 1.1.2 登录接口

##### 1.1.2.1 接口说明

- 接口地址：/api/user/login
- 请求方式：POST
- 参数格式：JSON（application/json）
- 接口描述：用户通过用户名 / 手机号 + 密码登录，返回登录态 token 及用户信息

##### 1.1.2.2 请求参数

| 参数名称 | 参数说明                    | 参数类型 | 是否必填 | 取值范围   | 示例值              |
| :------- | :-------------------------- | :------- | :------- | :--------- | :------------------ |
| account  | 登录账号（用户名 / 手机号） | string   | 是       | -          | jack123/13800138000 |
| password | 登录密码                    | string   | 是       | 6-16 位    | Jack@123456         |
| remember | 是否记住登录（7 天有效）    | boolean  | 否       | true/false | true                |

##### 1.1.2.3 请求参数样例

​```
{
  "account": "jack123",
  "password": "Jack@123456",
  "remember": true
}
​```

##### 1.1.2.4 响应参数

| 参数名称               | 参数说明 | 参数类型 | 是否必返 | 示例值                                   |
| :--------------------- | :------- | :------- | :------- | :--------------------------------------- |
| code                   | 响应码   | int      | 是       | 200                                      |
| msg                    | 响应信息 | string   | 是       | 登录成功                                 |
| data                   | 响应数据 | object   | 是       | -                                        |
| data.token             | 登录令牌 | string   | 是       | eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... |
| data.userInfo          | 用户信息 | object   | 是       | -                                        |
| data.userInfo.userId   | 用户 ID  | int      | 是       | 10001                                    |
| data.userInfo.nickname | 昵称     | string   | 是       | 杰克                                     |

##### 1.1.2.5 响应样例

###### 1.1.2.5.1 成功响应

​```
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userInfo": {
      "userId": 10001,
      "nickname": "杰克"
    }
  }
}
​```

###### 1.1.2.5.2 失败响应

​```
{
  "code": 401,
  "msg": "密码错误",
  "data": null
}
​```
```

## 格式规则

### 标题层级
- `###` 模块标题（如 `### 1.1 用户模块`）
- `####` 接口标题（如 `#### 1.1.1 获取用户信息`）
- `#####` 小节标题（接口说明、请求参数、响应参数等）
- `######` 子小节（成功响应、失败响应）

### 编号规则
- 模块编号：`1.1`, `1.2`, `2.1` ...
- 接口编号：`1.1.1`, `1.1.2` ...
- 小节编号：`1.1.1.1`（接口说明）, `1.1.1.2`（请求参数）, `1.1.1.3`（请求参数样例）, `1.1.1.4`（响应参数）, `1.1.1.5`（响应样例）
- 响应样例子节：`1.1.1.5.1`（成功响应）, `1.1.1.5.2`（失败响应）

### 表格对齐
- 所有列使用左对齐 `:-------`
- 列宽尽量对齐，提升可读性

### 请求参数表格列
`参数名称` | `参数说明` | `参数类型` | `是否必填` | `取值范围` | `示例值`

### 响应参数表格列
`参数名称` | `参数说明` | `参数类型` | `是否必返` | `示例值`

### 参数类型映射
| 代码类型 | 文档类型 |
|---------|---------|
| String / str | string |
| Integer / int / Long | int |
| Boolean / bool | boolean |
| Double / Float / float | number |
| List / Array / [] | array |
| Object / Map / dict / {} | object |
| Date / DateTime / LocalDateTime | string（日期时间） |

### GET 接口特殊处理
- 请求参数表格描述 query 参数（`?key=value`）
- 请求参数样例改为 URL 示例：`GET /api/user/info?userId=10001`
- 如果是 path variable，在接口地址中体现：`/api/user/{userId}`

### 无请求参数时
省略"请求参数"和"请求参数样例"两个小节，直接从"响应参数"开始，编号相应调整。
